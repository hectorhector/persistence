# persistence

Inspired by this Numberphile video:
https://www.youtube.com/watch?v=Wim9WJeDTHQ

This program uses the [GMP Library](https://gmplib.org/) to search for numbers with high Multiplicative Persistence. 

What is Multiplicative Persistence? Watch the video above, or check out this blog post: https://mathematrec.wordpress.com/2019/03/29/limited-persistence/.

Basically, you take a number in base 10, and multiply all the digits together. Then you take your new number and keep repeating the steps until you get  a 1 digit number. 
For example, take the number `93`: 
```
93 -> 9x3 = 27 
27 -> 2x7 = 14
14 - > 1x4 = 4
```
So, that took 3 steps, so 93 has a Multiplicative Persistence of 3!

The highest persistence ever found is 11. The lowest number with persistence 11 is: `277777788888899`. It is conjectured that no number in base 10 has a persistence higher than 11.

This program uses some tricks to search quickly through high-digit numbers with high persistence:
* Since the order of the numbers doesn't matter, many "duplicate" numbers may be skipped during the search. 
* Numbers with the digit 1 can be skipped, since it doesn't change a number's persistence. For example, if you have exhaustively searched searched all 2-digit numbers, the number `321` does not need to be checked because it has the same persistence as `32`.

After running this program, it appears that there are no numbers with more than 29 digits that have a multiplicity of 11 or higher. Once you get to 50-digit numbers, the best persistence you can get is 5. After writing this program, I agree with the conjecture that no number has a higher persistence than 11.

For each number of digits, it keeps track of what was the highest persistence found and how many numbers with that many digits have that persistence (not counting anagrams).
